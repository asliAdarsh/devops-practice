# Task 9: SSL/HTTPS — The Secure Tunnel

## Task Overview
Learn how to configure Nginx to terminate **SSL/TLS** connections and serve HTTPS traffic. This is essential for securing web applications with encryption, which is a requirement for modern web standards and SEO ranking.

## Key Concepts
- **SSL/TLS Termination** — Nginx decrypts incoming HTTPS traffic and forwards plain HTTP to the backend
- **Self-Signed Certificates** — Used for development/testing; production uses CA-signed certs (Let's Encrypt, etc.)
- **HTTP to HTTPS Redirection** — Redirects all HTTP traffic to the secure HTTPS endpoint
- **SSL Protocols & Ciphers** — Configuring which TLS versions and encryption ciphers to allow
- **HSTS (HTTP Strict Transport Security)** — Tells browsers to always use HTTPS

## Step-by-Step Instructions

### 1. Generate Self-Signed Certificates
```bash
mkdir -p certs
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout certs/server.key -out certs/server.crt \
  -subj "/CN=localhost"
```

### 2. Build and Start
```bash
cd task-9-ssl-https
docker compose up --build
```

### 3. Test HTTPS
```bash
# Access HTTPS (accept self-signed cert)
curl -k https://localhost

# Verify HTTP redirects to HTTPS
curl -v http://localhost
```

## Verification Steps
```bash
# Check SSL certificate details
openssl s_client -connect localhost:443 -servername localhost </dev/null

# Test SSL protocols
curl -k --tlsv1.2 https://localhost
curl -k --tlsv1.3 https://localhost

# Verify HTTP to HTTPS redirect
curl -v http://localhost 2>&1 | grep "301"
```

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Certificate path wrong in nginx.conf | Use absolute paths or verify relative to /etc/nginx/ |
| Permission denied on private key | Ensure private key has `chmod 600` permissions |
| Browser shows "Not Secure" | Self-signed certs will always show this. Use Let's Encrypt for production |
| Mixed content warnings | Ensure all assets (images, scripts) are loaded over HTTPS |
| Weak cipher configuration | Use modern TLS guidelines: `ssl_ciphers HIGH:!aNULL:!MD5;` |

## Best Practices
1. **Use Let's Encrypt for production** — Free, automated SSL certificates via Certbot
2. **Enable HSTS** — Add `add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;`
3. **Disable old TLS versions** — Only allow TLSv1.2 and TLSv1.3
4. **Redirect all HTTP to HTTPS** — Prevents unencrypted access
5. **Renew certificates before expiry** — Set up automated renewal with cron or systemd timers

## Success Criteria
- [ ] HTTPS server responds on port 443
- [ ] HTTP on port 80 redirects to HTTPS (301)
- [ ] Certificate is presented correctly during TLS handshake
- [ ] Backend receives proxied requests with correct headers
- [ ] SSL protocols TLSv1.2 and TLSv1.3 are enabled
- [ ] Weak ciphers and SSLv3/TLSv1.0/1.1 are disabled
